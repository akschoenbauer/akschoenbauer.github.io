Challenge 5: Accessibility Concerns

While working on Challenge 2, I began to wonder how a screen reader would approach the chessboard and chess piece code on this webpage. In lecture, it has been mentioned that screenreaders often don't read tables correctly since the formatting confuses them. Since the chessboard is created from a table format, I thought that may be an issue here. I also am not sure how a screen reader would approach the unicode chess pieces that are added to the board. To address the trouble the screenreader may have with the chessboard, I wondered if there was a way to add alternative text for it, like you would for an image. If it is not possible to add alternate text to the table, the issue could also be fixed by screenshotting the chessboard from the live server preview, and then uploading it to the site as an image with alternative text.

I also wondered how a form would be read by a screenreader - would the reader prompt the user to answer the form fields by posing the quetions asked? This is another section where alternative text could be helpful. I also know that when you inspect a webpage, you can see the comments left in the HTML code. I wondered if leaving comments/pseudocode could be another way to provide alternative text to someone who has a sight disability.

Overall, this assignment has made me curious about other ways to make webpages accessbile to people of all abilities. I hope to learn more either in class or through my own research about various techniques and practices to employ in order to do so.
